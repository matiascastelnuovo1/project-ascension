# Project ascension


## Uso de Branches

Todos los branches deben seguir esta convención de nombres:

- `feature/descripcion` — para features nuevas
- `fix/descripcion` — para bugfixes

Ejemplos: `feature/player-movement`, `fix/collision-bug` - SIEMPRE todo en miniscula y separando palabra por palabra con guion del medio

No se puede pushear directo a `main`. Todo cambio entra por Pull Request.
